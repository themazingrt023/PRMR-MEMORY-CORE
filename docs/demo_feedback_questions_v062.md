# PRMR Memory Core V0.62 Demo Feedback Questions

Use these after a controlled demo walkthrough. The goal is learning, not selling.

## Core Understanding

1. Did the problem make sense?
2. Could you explain PRMR Memory Core back in your own words?
3. Did continuity feel different from summaries, vector search, or ordinary storage?
4. Which part of the demo made the idea clearest?
5. What confused you?

## Use Case Fit

6. Which use case felt strongest: AI agent memory, customer support continuity, SaaS user-history continuity, education progress continuity, legal/research continuity, fintech/risk synthetic sandbox, or company knowledge continuity?
7. Which use case felt weakest or least credible?
8. Would you want to test this on synthetic data?
9. What workflow would you use for a synthetic test?
10. What would make this credible enough for a future pilot discussion?

## Trust And Boundary Feedback

11. Were the local controlled-alpha boundaries clear?
12. Did any part sound like it was claiming production readiness or external validation?
13. Were the public-safe explanations cautious enough?
14. Did the least-harm action language avoid final automated decisions?
15. What risks or missing pieces do you see?

## Product Feedback

16. What output would you want to see first in a real workflow?
17. What would you need from docs or developer examples?
18. What would make the demo more convincing without overclaiming?
19. What would make you hesitate?
20. Who else should review this controlled demo?

## Follow-Up Notes

- Do not collect real sensitive data in this feedback process.
- Do not promise live API access.
- Do not promise API keys.
- Do not promise production use.
- Do not promise certification.
- Record feedback manually and keep it separate from public-safe reports.
