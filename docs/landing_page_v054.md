## Problem

Modern systems collect long histories but often lose the shape of change. AI agents, support teams, and risk workflows can inherit stale context, miss the last meaningful turn, or carry too much raw history into moments that need a smaller review-safe packet.

The problem is not simply storage. It is continuity: what changed, which signals are still active, what became stale, what evidence is missing, and what needs review before action.

## Solution

PRMR Memory Core compresses event histories into continuity packets. A packet is designed to preserve the useful shape of change while reducing raw noise. It can carry current state, active signals, stale signals, evidence summaries, and a review boundary.

The system is currently demonstrated through local controlled-alpha evidence and synthetic demo fixtures.

## API Flow

1. Ingest synthetic or approved events.
2. Generate a continuity packet for a scoped client, vault, and namespace.
3. Reconstruct current memory state from the packet and stored events.
4. Generate a public-safe explanation.
5. Return a least-harm action boundary for review support.
6. Fetch a public-safe report for the owner.
7. Deny wrong-key or cross-client access attempts.

Public API flow copy should describe backend-mediated access only. The page must not place credentials in browser code or ask users to paste credentials into public demos.

## Use Cases

AI agent memory continuity: Preserve what changed across sessions without replaying every raw event into the next turn.

Customer support continuity: Carry forward the current state of a user-history thread while separating active signals from stale notes.

Risk continuity sandbox: Track review-relevant changes without accusation language or final punitive decisions.

Operations knowledge continuity: Keep compact state packets for workflows where old context can quietly become misleading.

## Why Continuity Matters

Storage can preserve everything and still fail to preserve meaning. Continuity helps a system understand the difference between raw history and current state.

For AI systems, continuity reduces context drift. For organisations, continuity supports safer handoffs, clearer review, and less reliance on brittle summaries.
