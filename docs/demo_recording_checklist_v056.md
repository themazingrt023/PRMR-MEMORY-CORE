# PRMR V0.56 Demo Recording Checklist

Company: Afternum Industries  
Product: PRMR Memory Core  
Version: V0.56

## Before Recording

- Local frontend server is running.
- `http://localhost:3000/demo` loads.
- Scenario selector is visible.
- `Run Local Demo` button is visible.
- The page shows the boundary: synthetic data only, local controlled-alpha demo, not hosted production.
- Browser zoom is set to 90-100 percent.
- Recording resolution is 1920x1080 or 2560x1440.
- Frame rate is 30 fps or 60 fps.
- Cursor is visible but not distracting.
- Notifications are off.
- Browser bookmarks and personal tabs are hidden.
- Terminal windows with paths, keys, or private traces are not visible.

## During Recording

- Select one synthetic scenario.
- Click `Run Local Demo`.
- Wait for result cards to populate before speaking through them.
- Show synthetic events.
- Show continuity packet.
- Show reconstructed state.
- Show public-safe explanation.
- Show least-harm action.
- Show report preview.
- Show denial path proof.
- Keep the explanation calm and direct.
- Keep the demo under five minutes.

## Safety Checks

- No raw keys visible.
- No private/internal report is shown.
- No private diagnostic packet is shown.
- No real sensitive data is used.
- No hosted backend is claimed.
- No production readiness is claimed.
- No bank approval is claimed.
- No compliance approval is claimed.
- No legal approval is claimed.
- No external security certification is claimed.
- No real-world validation is claimed.
- No final automated decision is claimed.

## Recommended Recording Flow

1. Start on the homepage for a brief brand moment.
2. Navigate to `/demo`.
3. Say the opening line.
4. Explain the problem.
5. Select a scenario.
6. Click `Run Local Demo`.
7. Walk through the cards from top to bottom.
8. Mention the internal/local evidence summary.
9. Say the boundary statement.
10. Close with the controlled alpha conversation CTA.

## Visual Polish Notes

- Move the cursor deliberately.
- Pause after clicking the run button.
- Keep the demo page centered.
- Avoid scrolling too quickly.
- Use one scenario for short recordings.
- Use all three scenarios only for longer founder walkthroughs.
- End on the boundary or report preview, not on a blank section.
