# PRMR V0.56 Demo Claim Boundaries

Company: Afternum Industries  
Product: PRMR Memory Core  
Version: V0.56

## Purpose

This document keeps founder demo language honest. It separates what can be said in a screen-recorded local demo from what must wait for later validation.

## Allowed Claims

- PRMR Memory Core has a local controlled-alpha demo.
- The demo uses synthetic data only.
- The demo connects the frontend to a local server-side demo bridge.
- The local demo renders continuity packet output.
- The local demo renders reconstructed state output.
- The local demo renders public-safe explanation output.
- The local demo renders least-harm action output.
- The local demo renders a public-safe report preview.
- The local demo shows wrong-key and cross-client denial outcomes.
- Internal tests passed for the referenced local milestones.
- The sandbox is API-shaped.
- Public-safe reports are separated from private diagnostics in the local demo flow.

## Forbidden Claims

Do not claim:

- production-ready
- hosted API live
- bank approved
- compliance approved
- legal approval
- external security certification
- real-world validated
- guaranteed fraud detection
- final automated decisions

## Safe Boundary Statement

This is local controlled-alpha evidence using synthetic data. It is not hosted production, not bank approval, not compliance approval, not legal approval, not external security certification, and not real-world validation.

## Safer Substitutions

- Say `local controlled-alpha demo`, not `live production system`.
- Say `synthetic data`, not `real customer data`.
- Say `internal/local checks passed`, not `externally validated`.
- Say `review-oriented next step`, not `final automated decision`.
- Say `public-safe report preview`, not `full internal report`.
- Say `access denial outcome`, not `security certification`.

## What To Avoid On Camera

- Do not show terminals containing secret material.
- Do not show private/internal reports.
- Do not show diagnostic packet traces.
- Do not describe the local bridge as a hosted backend.
- Do not imply banking, regulatory, legal, or external security approval.
- Do not imply real-world deployment.
